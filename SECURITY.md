# Security Policy

## Supported Versions

Only the latest release on the `main` branch is actively supported with security
updates.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

To report a vulnerability:

1. Preferred: use [GitHub private vulnerability reporting](https://github.com/jerdaw/healtharchive-datasets/security/advisories/new).
2. Alternative: email `security@healtharchive.ca`.

## Scope

The following are in scope for security reports:

- Dataset generation and validation scripts
- Data integrity mechanisms
- Dependencies with known vulnerabilities
- Accidental exposure of non-public data in release artifacts

The following are out of scope:

- Production hosting infrastructure
- Denial-of-service attacks
- Social engineering

## Disclosure

Confirmed vulnerabilities should be fixed before public disclosure whenever
practical. Public disclosure should remain public-safe and avoid credentials,
private operational details, internal hostnames, private IPs, and personal
contact information.
